# Setup

End-to-end deployment of the Comment Triage Agent. Realistic time: 30–45 minutes if you've never used n8n or Supabase, 10–15 if you have.

## Prerequisites

- A self-hosted n8n instance (Docker, npm `npx n8n`, or Render free tier). Cloud n8n works too but credentials handling differs slightly.
- A YouTube account with at least one public video (for testing the workflow against real comments).

You'll also create accounts on the way for Anthropic, Supabase, and Slack — all free to start.

## 1. Import the workflow

1. In n8n, open the workflow editor.
2. Top-right menu → **Import from File** → select `workflow.json` from this repo.
3. n8n loads the workflow but flags it: every credential is unbound and every placeholder still says `YOUR_*`. That's normal — you'll fix them in the next steps.

## 2. Create the Supabase project and table

1. Go to [supabase.com](https://supabase.com) → **Start your project** → sign in with GitHub.
2. **New Project**:
   - Name: anything (e.g. `youtube-comment-triage`)
   - Database Password: generate a strong one, save it in a password manager
   - Region: pick the one closest to your n8n instance
   - Plan: **Free**
3. Wait ~2 minutes for provisioning.
4. Sidebar → **SQL Editor** → **New query** → paste and run:

```sql
create table comment_classifications (
  id bigint generated by default as identity primary key,

  -- YouTube identifiers
  comment_id text not null unique,
  thread_id text not null,
  video_id text not null,

  -- AI classification
  category text not null check (category in ('spam', 'FAQ', 'feedback', 'sponsor', 'other', 'acknowledge')),
  confidence numeric(3, 2),
  reasoning text,

  -- FAQ reply draft (NULL for non-FAQ rows)
  reply_draft text,
  reply_confidence numeric(3, 2),
  reply_rationale text,
  reply_generated_at timestamptz,

  -- Original comment snapshot
  text text not null,
  text_html text,
  author text,
  author_channel_id text,
  author_channel_url text,
  like_count integer,
  total_replies integer,
  published_at timestamptz,

  -- Audit
  classified_at timestamptz default now()
);

create index idx_classifications_video on comment_classifications(video_id);
create index idx_classifications_category on comment_classifications(category);
create index idx_classifications_classified_at on comment_classifications(classified_at desc);
```

5. Verify in **Table Editor** that `comment_classifications` exists with 0 rows.
6. Sidebar → **Project Settings** → **API**. Copy these two values:
   - **Project URL** (e.g. `https://xxxxxx.supabase.co`)
   - **Project API key → `service_role` (secret)** — needed for inserts. The `anon` key does not have write permissions.

## 3. Get the YouTube Data API key

1. Go to [Google Cloud Console](https://console.cloud.google.com).
2. Create a new project (or pick an existing one).
3. **APIs & Services → Library** → search **YouTube Data API v3** → **Enable**.
4. **APIs & Services → Credentials** → **Create credentials** → **API key**.
5. Copy the key. The free quota (10,000 units/day) covers thousands of comment polls.

## 4. Get the Anthropic API key

1. Sign up at [console.anthropic.com](https://console.anthropic.com).
2. Add at least $5 of credit if you want Tier 2 (recommended to avoid rate limits on initial testing). Free trial credit is fine for first runs.
3. **API Keys → Create Key** → copy it.

## 5. Create 3 Slack incoming webhooks

You need three separate webhook URLs, one per channel. Pick or create three Slack channels first:

- `#ops` — receives the digest of feedback + other comments
- `#biz` — receives the digest of sponsorship inquiries
- `#ops` (or a dedicated `#faq` channel) — receives FAQ reply drafts, one per FAQ

For each channel:

1. Go to your Slack workspace → **Apps** → search **Incoming Webhooks** → **Add to Slack**.
2. Pick the target channel → **Add Incoming Webhooks integration**.
3. Copy the **Webhook URL** (format: `https://hooks.slack.com/services/T.../B.../...`).

If you don't want a dedicated app, you can also use Slack's [legacy webhook flow](https://api.slack.com/messaging/webhooks).

## 6. Configure credentials in n8n

In n8n, sidebar → **Credentials** → **New**:

**Anthropic API**:
- Type: `Anthropic`
- API Key: the key from step 4

**Supabase API**:
- Type: `Supabase`
- Host: the Project URL from step 2
- Service Role Secret: the `service_role` key from step 2

The YouTube API key and the Slack webhook URLs are passed directly in node parameters (next step), not as n8n credentials.

## 7. Replace the placeholders in the workflow

Open the workflow in n8n and edit these nodes:

| Node | Field | Replace with |
|---|---|---|
| `HTTP Request` (YouTube) | Query parameter `key` | Your YouTube Data API key |
| `HTTP Request` (YouTube) | Query parameter `videoId` | The ID of the video you want to test against (the part after `?v=` in the URL) |
| `slack FAQ` | URL | Your FAQ webhook URL |
| `Slack ops` | URL | Your `#ops` webhook URL |
| `Slack biz` | URL | Your `#biz` webhook URL |
| `Classify comment` (AI Agent) | Anthropic Chat Model sub-node → Credential | Select the Anthropic credential |
| `Generate FAQ reply` (AI Agent) | Anthropic Chat Model1 sub-node → Credential | Select the Anthropic credential |
| `Get processed IDs`, `Get current run records`, `Create a row` (Supabase) | Credential | Select the Supabase credential |

Save the workflow.

## 8. First test run

1. Click the **Manual Trigger** node at the top-left.
2. Top-right → **Test Workflow** (or `Ctrl+Enter`).
3. Watch the nodes turn green sequentially. Expected duration: 4–6 minutes for ~100 comments (the Wait nodes are intentional, see the README's "Production considerations" section).
4. Check:
   - Slack `#ops` should receive a digest with the feedback + other comments
   - Slack `#biz` should receive a sponsor digest if any sponsor comments were detected
   - Slack FAQ should receive one card per FAQ with a draft reply
   - Supabase `comment_classifications` should have ~100 new rows

Query to inspect the distribution:

```sql
select category, count(*) 
from comment_classifications 
group by category 
order by count(*) desc;
```

A healthy run looks roughly like 60–75% `acknowledge`, 10–20% `feedback`, 5–10% `FAQ` + `other`, 0–5% `sponsor` + `spam`. Numbers vary heavily by channel.

## 9. Optional — enable scheduled runs

Once the manual run produces what you expect:

1. Replace the **Manual Trigger** with a **Schedule Trigger** (every 30 minutes is a sane default).
2. Re-enable the **Get processed IDs** node (it's disabled by default in the demo). This activates the dedup — subsequent runs only classify new comments.
3. Toggle the workflow to **Active** in the top-right corner.

The workflow now polls the video every 30 minutes, classifies only what's new, and posts a digest to Slack only when there are items worth attention.

## Troubleshooting

**"Rate limit exceeded" from Anthropic** — you're on Tier 1 (30K input tokens/min). Either upgrade to Tier 2 (add $5 of credit at [console.anthropic.com/settings/billing](https://console.anthropic.com/settings/billing)) or lower the Loop Over Items batch size from 15 to 10.

**Supabase insert error `All object keys must match`** — the Parse classification node must output `reply_*: null` for non-FAQ rows. Check the node's code is the version in this repo (the normalized-schema variant).

**Slack returns 400** — usually a malformed JSON body. Most often caused by user-generated text (comment body, author name) containing unescaped quotes or newlines. The Slack node payloads in `workflow.json` already use `JSON.stringify(...)` defensively — make sure no copy-paste accidentally removed those.

**The digest is empty even though Supabase has rows** — the `filter recent` node uses `$execution.startedAt` (or a 15-min cutoff in older versions). Make sure rows in DB have `classified_at` populated and that the timestamp is fresh.

**`Referenced node doesn't exist` errors** — a Code node references another node by name (e.g. `$('Parse classification').item.json`). After renaming any node, update the references in downstream Code nodes — n8n doesn't auto-rename them.
